# Roadmap

## Фаза 0 — Injector + DLL bootstrap
**Статус:** partial (Injector — completed, DLL runtime bootstrap — partial)

### Цели
- Гарантированная доставка DLL в процесс `Wow.exe`.
- Минимальный безопасный startup внутри `DllMain`.

### Фактическое состояние
- ✅ Injector реализован и закрывает базовый happy-path инжекта.
- ✅ `DllMain` создаёт отдельный поток для инициализации.
- ⏳ Core lifecycle пока содержит заглушки и требует реального orchestration.

---

## Фаза 1 — Hooks + offsets wiring
**Статус:** planned / in-progress

### План
- Реализовать установку/снятие EndScene hook.
- Подключить `offsets.h` к реальным адресам/сигнатурам.
- Добавить безопасный tick-контур в игровом потоке.
- Ввести диагностику: лог установки hook, watchdog на ошибки.

### Результат фазы
- Стабильный игровой цикл, от которого можно питать managers/lua/ipc.

---

## Фаза 2 — Object iteration + Lua engine
**Статус:** planned

### План
- Реализовать чтение object list и нормализованную типизацию сущностей.
- Подготовить `ObjectManager::Update()` с кэшированием/валидаторами.
- Расширить Lua engine до диспетчера команд, исполняемых в игровом потоке.
- Добавить базовые сценарии: target select, spell cast, movement command.

### Результат фазы
- Рабочий доменный слой взаимодействия с миром и внутриигровыми командами.

---

## Фаза 3 — Managers / Products / IPC
**Статус:** planned

### План
- Довести менеджеры (`movement`, `spells`, `inventory`, `target`) до рабочих сервисов.
- Сформировать продуктовые пайплайны (например, gatherer loop) поверх менеджеров.
- Подключить IPC сервер с протоколом команд и статусной телеметрией.
- Подготовить интеграцию с GUI-клиентом (управление профилями/режимами).

### Результат фазы
- Управляемая извне, расширяемая и практически применимая runtime-система.
